?? CORE FLOW
?????????????????????????????????????????????????????????????????????????????????

Player Action
    ?
Hazard.Mark() / HazardTrigger.OnTriggerEnter() / IXRInteractable method
    ?
EventPublisher.Publish*() call
    ?
MqttEventClient.PublishEvent() ? Async queue
    ?
Background Task publishes to MQTT broker
    ?
Backend receives on "events/" topic
    ?
Backend calculates score, stores in PostgreSQL
    ?
UI calls EventPublisher.RequestFinalScore()
    ?
Backend publishes to "response/points"
    ?
MqttEventClient receives ? Main thread processes
    ?
MqttEventClient.OnScoreResponseReceived fires
    ?
TrainingUIManager.HandleScoreResponse() updates UI


?? QUICK API
?????????????????????????????????????????????????????????????????????????????????

TrainingScenarioController
  - StartScenario()               ? Publishes SESSION_START
  - EndScenario()                 ? Publishes SESSION_END + requests final score
  - ResetScenario()               ? Resets state for new run

EventPublisher
  - PublishSessionStarted(scenarioName)
  - PublishSessionEnded(finalScore, duration)
  - PublishActionInteract(targetObjectId, action, toolUsed, isSuccess, reason, consequence)
  - PublishHazardMarked(hazardId, correct, points)
  - PublishHseAlert(triggerId, description, penalty)
  - RequestFinalScore()

MqttEventClient
  - PublishEvent(TrainingEvent evt)
  - RequestFinalScore(sessionId)
  - IsConnected property
  - OnScoreResponseReceived event

Hazard
  - Mark()                        ? Call when player identifies hazard
  - Reset()                       ? Called on scenario reset

HazardTrigger
  - OnTriggerEnter(Collider)     ? Auto-fires HSE_ALERT_EVENT
  - Reset()                       ? Called on scenario reset

IXRInteractable
  - ObjectId property
  - OnInteractSuccess(action, toolUsed)
  - OnInteractFailure(action, toolUsed, reason, consequence)


?? EVENT TYPES & TOPICS
?????????????????????????????????????????????????????????????????????????????????

PUBLISH TO "events/" topic:
  - SESSION_START         (scenario begins)
  - SESSION_END           (scenario completes)
  - HAZARD_MARKED         (player identified hazard)
  - HSE_ALERT_EVENT       (safety violation triggered)
  - ACTION_INTERACT       (custom interaction event)

PUBLISH TO "request/points" topic:
  - SCORE_REQUEST         (query final score from backend)

SUBSCRIBE TO "response/points" topic:
  - Score responses       (final_score from backend)

?? IMPORTANT NOTES
?????????????????????????????????????????????????????????????????????????????????

? NO local score tracking - backend calculates from events
? ALL MQTT operations are async - main thread never blocks
? Events fire ONCE per session (one-shot) - prevented by _marked flag + cooldown
? Only request final score via EventPublisher.RequestFinalScore()
? UI should only listen to OnScoreResponseReceived for authoritative score
