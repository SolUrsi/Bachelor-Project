DELETED FILES (no longer needed):

CREATED/REPLACED FILES:

1. Mqtt/EventModel.cs ?
   - Contains all serializable event classes
   - EventHeader, EventPayload, TelemetryData
   - TrainingEvent, ScoreRequestMessage, ScoreResponseMessage
   - Supports new metadata field for ACTION_INTERACT events

2. Mqtt/MqttEventClient.cs ? (NEW)
   - Paho M2MQTT-based MQTT client
   - Async-only operation (never blocks main thread)
   - Publishes to: "events/" and "request/points" topics
   - Subscribes to: "response/points" topic
   - Thread-safe concurrent queues for main?background communication
   - Automatic reconnect with exponential backoff
   - Event OnScoreResponseReceived for UI to receive final scores
   - Conditional compilation: only active if MQTT_ENABLED symbol is defined

3. Mqtt/EventPublisher.cs ? (REWRITTEN)
   - Simplified to: build TrainingEvent + queue for publishing
   - NO MQTT details (MqttEventClient owns transport)
   - Public methods:
     * PublishSessionStarted(scenarioName)
     * PublishSessionEnded(finalScore, duration)
     * PublishActionInteract(targetObjectId, action, toolUsed, isSuccess, reason, consequence)
     * PublishHazardMarked(hazardId, correct, points)
     * PublishHseAlert(triggerId, description, penalty)
     * RequestFinalScore()
   - Builds event header with timestamp, sessionId, sceneId, eventType

4. TrainingStateMachine.cs ? (SIMPLIFIED)
   - States: NotStarted ? Active ? Ended
   - Used for local gameplay flow only
   - Guards when events should fire (IsActive property)
   - NO event publishing; purely tracks state

5. Hazard.cs ? (REWRITTEN)
   - Publishes HAZARD_MARKED events when marked
   - One-shot + cooldown to prevent spam
   - Calls EventPublisher.PublishHazardMarked() directly
   - No local score tracking

6. HazardTrigger.cs ? (REWRITTEN)
   - Publishes HSE_ALERT_EVENT on OnTriggerEnter
   - One-time + cooldown + tag filtering
   - Calls EventPublisher.PublishHseAlert() directly

7. TrainingScenarioController.cs ? (REWRITTEN)
   - Main coordinator for scenario lifecycle
   - Creates SessionContext and initializes EventPublisher
   - Methods: StartScenario(), EndScenario(), ResetScenario()
   - Publishes SESSION_START on StartScenario()
   - Publishes SESSION_END and requests final score on EndScenario()
   - No local score tracking; no event batching

8. TrainingUIManager.cs ? (REWRITTEN)
   - Listens ONLY to MqttEventClient.OnScoreResponseReceived
   - Displays authoritative final score from backend
   - No event publishing; UI is read-only

9. IXRInteractable.cs ? (NEW)
   - Interface for custom interactive game objects
   - Methods: OnInteractSuccess(), OnInteractFailure()
   - Implement to publish ACTION_INTERACT events

10. ARCHITECTURE.txt ? (NEW)
    - 300+ line comprehensive setup and design guide
    - Event flow examples
    - Installation instructions
    - Key design principles
    - Example custom interactable implementation

SETUP INSTRUCTIONS
???????????????????????????????????????????????????????????????????????????????

1. INSTALL PAHO M2MQTT LIBRARY
   ?????????????????????????????
   Option A (Recommended): NuGet Package Manager
     PM> Install-Package uPLibrary.Networking.M2Mqtt
     
   Option B: Manual DLL
     1. Download M2Mqtt.dll from NuGet.org
     2. Place in: Assets/Plugins/
     3. Set Inspector: CPU = Any, OS = Any

2. ENABLE MQTT_ENABLED DEFINE SYMBOL
   ???????????????????????????????????
   Unity Editor:
     - Edit ? Project Settings ? Player
     - Other Settings ? Scripting Define Symbols
     - Add: MQTT_ENABLED
     - Platform: Standalone (or all platforms you support)

3. SETUP SCENE HIERARCHY
   ??????????????????????
   In your Main Scene:
     GameObject: "TrainingManager"
       ? MqttEventClient (component)
       ? EventPublisher (component)
       ? TrainingScenarioController (component)
          ? Reference: MqttEventClient (above)
          ? Reference: EventPublisher (above)

   Add to scene alongside other game controllers (Hazard, HazardTrigger objects, etc.)

4. CONFIGURE MQTT BROKER
   ??????????????????????
   In MqttEventClient Inspector:
     - Broker Host: "your.vps.ip.address" (e.g., "157.245.79.217")
     - Broker Port: 1883
     - Client ID: traftec-vr (auto-appended with device ID)
     - Topics:
       * Events: "events/"
       * Request Points: "request/points"
       * Response Points: "response/points"

5. WIRE UP UI BUTTONS
   ??????????????????
   "START" Button:
     OnClick ? TrainingScenarioController.StartScenario()
   
   "END" / "SUBMIT" Button:
     OnClick ? TrainingScenarioController.EndScenario()

   Score Display (optional):
     Add TrainingUIManager to Canvas
     Wire UI elements in Inspector

6. CREATE HAZARDS
   ??????????????
   For each hazard in scene:
     1. GameObject with collider
     2. Add Hazard component
     3. Set hazardId (e.g., "H-001")
     4. Set isCorrectHazard: true/false
     5. Wire XR interaction to Hazard.Mark()

   For each trigger:
     1. GameObject with Collider (IsTrigger=true)
     2. Add HazardTrigger component
     3. Set triggerId, description, penalty
     4. Set onlyOnce=true (optional)

7. TEST IN EDITOR
   ??????????????
   - Play scene
   - Check Console for: "[MQTT] Connected to ..."
   - Press START button ? should see SESSION_START queued
   - Interact with hazards ? should see HAZARD_MARKED events
   - Press END button ? should see SESSION_END + score request
   - If backend responds, UI updates with final score

EXAMPLE 1: Publishing an interaction event
????????????????????????????????????????

public class BreechBlock : MonoBehaviour, IXRInteractable
{
    public string ObjectId => "breech_block_01";

    public void OnInteractSuccess(string action, string toolUsed = "none")
    {
        EventPublisher.Instance?.PublishActionInteract(
            targetObjectId: ObjectId,
            action: action,
            toolUsed: toolUsed,
            isSuccess: true
        );
    }

    public void OnInteractFailure(string action, string toolUsed, string reason, string consequence = "")
    {
        EventPublisher.Instance?.PublishActionInteract(
            targetObjectId: ObjectId,
            action: action,
            toolUsed: toolUsed,
            isSuccess: false,
            reason: reason,
            consequence: consequence
        );
    }

    public void AttemptOpen(string toolUsed)
    {
        if (toolUsed == "insulated_screwdriver")
            OnInteractSuccess("open", toolUsed);
        else
            OnInteractFailure("open", toolUsed, "missing_tool", "electric_shock");
    }
}

EXAMPLE 2: Publishing from a custom hazard detector
??????????????????????????????????????????????????

public class NoSafetyGear : MonoBehaviour
{
    private void OnTriggerStay(Collider other)
    {
        if (other.CompareTag("Player"))
        {
            // One-time event to prevent spam
            if (!_reported)
            {
                EventPublisher.Instance?.PublishHseAlert(
                    triggerId: "no-safety-gear",
                    description: "Entered restricted area without protective equipment",
                    penalty: 25
                );
                _reported = true;
            }
        }
    }

    private bool _reported;
}

IMPORTANT NOTES
???????????????????????????????????????????????????????????????????????????????

?? MQTT LIBRARY REQUIREMENT
   The code will compile without M2MQTT installed, but:
   - Add MQTT_ENABLED define symbol only if library is installed
   - Without the symbol, MqttEventClient will log a warning but won't break
   - For deployment to Meta Quest 3, ensure M2MQTT is compiled into APK

?? NO LOCAL SCORE TRACKING
   The game NEVER tracks score locally. Every event is published:
   - HAZARD_MARKED: +points for correct, -penalty for wrong
   - HSE_ALERT_EVENT: -penalty for safety violation
   Backend calculates authoritative final_score from these events.
   UI only displays score when received from backend via response/points.

?? ONE-SHOT EVENTS
   Hazards and triggers are designed to fire once per session:
   - Hazard._marked flag prevents re-marking
   - Cooldown system prevents physics glitch spam
   - Set cooldownSeconds in Inspector (default 1s)

?? THREAD SAFETY
   - All MQTT operations run in background Task threads
   - Main thread queues events in ConcurrentQueue
   - Background thread dequeues and publishes
   - Score responses queued by MQTT thread, processed by main thread Update()
   - Queues are thread-safe; no locks needed

TROUBLESHOOTING
???????????????????????????????????????????????????????????????????????????????

? "MqttEventClient not found" at runtime
   ? Ensure MqttEventClient component is in scene or prefab
   ? Check if MQTT_ENABLED symbol is defined
   ? Verify M2MQTT DLL is in Assets/Plugins/

? Events not publishing
   ? Check Console for "[MQTT] Connected to ..." message
   ? Verify broker host/port are correct
   ? Check if network can reach VPS on port 1883
   ? Ensure firewall allows outbound MQTT

? Score never updates in UI
   ? Did you call TrainingScenarioController.EndScenario()?
   ? Is backend running and subscribed to "events/" topic?
   ? Check MQTT broker logs for "request/points" messages
   ? Verify backend is publishing to "response/points" topic

? Hazards fire multiple times
   ? Increase cooldownSeconds in HazardTrigger Inspector
   ? Check if isTracking logic is correct
   ? Verify collider isn't set to multiple calls per frame
